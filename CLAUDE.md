## 개발 방식

### TDD
- 구현 코드보다 테스트를 먼저 작성한다 (Red → Green → Refactor)
- 테스트가 실패하는 것을 확인한 후 구현한다
- 한 번에 하나의 테스트만 통과시킨다
- 테스트를 통과하는 최소한의 코드만 작성한다

### 명령어 및 플러그인
- bash 명령어 사용을 지양함
- bash 명령어를 사용하거나 플러그인을 사용 시 어떤 행위인지 설명하고 반드시 허락을 받아야 함

### 파일

- 요청하지 않은 파일은 조회 시 허락을 받아야 함

### 코드 컨벤션

- 모든 자바 코드는 카멜케이스 사용
- ASCII 코드 외의 코드 사용 금지
